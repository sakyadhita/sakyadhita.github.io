# Plan: Fix Broken Links in Conference Slideshows

## Objective

Resolve the broken image links on individual conference pages by generating optimized image URLs for the conference slideshow components, matching the logic currently implemented in the main conference listing layout.

## Key Files & Context

- `src/pages/conferences/[...slug].astro`: The dynamic route for individual conference pages.
- `src/layouts/ConfLayout.astro`: The layout for the main `/conferences` page (used as a reference for the correct implementation).

## Implementation Steps

1. **Import Dependencies:** Add the necessary imports for image processing and markdown parsing in `src/pages/conferences/[...slug].astro`:
   ```javascript
   import { getImage } from 'astro:assets'
   import { marked } from 'marked'
   import { getRawImage } from '../../lib/images'
   ```
2. **Process Conference Data:** Update the data fetching logic in `src/pages/conferences/[...slug].astro` to map over the raw `conference` collection. For each conference:
   - Parse the markdown body using `marked.parse()`.
   - Iterate over `c.data.slideShowImages`, mapping each raw URL via `getRawImage()`, and then process it with `getImage({ src: raw, format: 'webp', width: 1200 })` to generate an optimized URL.
   - Attach the parsed `htmlBody` and `optimizedImages` arrays back to the conference object.
3. **Pass Processed Data:** Ensure the fully processed and reversed data array is passed down to the `ConferenceDesktop` and `MobileConference` components.

## Verification & Testing

- Navigate to individual conference pages (e.g., `/conferences/19` and `/conferences/18`) and verify that the hero slideshow images load correctly.
- Ensure the images rendered use the optimized `.webp` format paths generated by Astro's `_image` endpoint.
- Run `pnpm test:e2e` to ensure no regressions occur.
